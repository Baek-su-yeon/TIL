## DDL (CREATE, DROP, ALTER)
- 데이터의 기본 구조 및 형식 변경

    ### CREATE
    ```SQL
    CREATE TABLE table_name(
        -- 컬럼명, 데이터 타입, 필드 제약조건
        column_1 data_type constrains,
        column_2 data_type constrains,
    )
    ```
    - 데이터 타입
        
        |타입|설명|
        |:---|:---|
        |NULL|아무런 값도 포함하지 않음|
        |TEXT|문자열|
        |INTEGER|정수|
        |REAL|부동소수점|
        |BLOB|이미지, 동영상, 문서 등 바이너리 데이터|
    
    - 제약 조건

        |명령어|역할|
        |:---|:---|
        |PRIMARY KEY|해당 필드를 기본 키로 지정(INTEGER타입만 가능)|
        |NOT NULL|해당 필드에 NULL 값 허용 ❌|
        |FOREIGN KEY|다른 테이블과의 외래 키 관계를 정의|
    
    ### ALTER
    - ADD: 필드 추가 (🚨 한번에 여러 필드 추가 ❌, 하나씩 해야함)
        ```SQL
        ALTER TABLE table_name ADD COLUMN column_definition;

        -- 예시
        ALTER TABLE example ADD COLUMN Country VARCHAR(100) NOT NULL DEFAULT 'default value';
        ```
    - RENAME: 필드 이름 변경
        ```SQL
        ALTER TABLE table_name RENAME COLUMN current_name TO new_name;
        ```
    - DROP: 지정 필드 삭제
        ```SQL
        ALTER TABLE table_name DROP COLUMN column_name;
        ```
    - RENAME TO: 테이블 이름 변경
        ```SQL
        ALTER TABLE table_name RENAME TO new_table_name;
        ```
    
    ### DROP
        ```SQL
        DROP TABLE table_name;
        ```

## DML (INSERT, UPDATE, DELETE)
- 데이터 조작
    
    ### INSERT (테이블 레코드 삽입)
        ```SQL
        INSERT INTO table_name (column1, column2, ...) -- 테이블 이름과 필드 목록
        VALUES (value1, value2, ...) -- 해당 필드에 삽입할 값 목록

        -- 예시
        INSERT INTO
	    articles (title, content, createdAt)
        VALUES
	    ('title1', 'content1', '1900-01-01'),
	    ('title2', 'content2', '1900-01-01'),
	    ('title3', 'content3', DATE()); -- 오늘 날짜
        ```
    ### UPDATE (테이블 레코드 수정)
        ```SQL
        UPDATE table_name
        SET column_name = value, -- 수정할 필드 = 새 값
        [WHERE condition]; -- 수정할 레코드 지정 조건, 없으면 모든 레코드 수정

        -- 예시: articles 테이블 1번 레코드의 title 필드 값을 'update Title'로 변경
        UPDATE articles SET title = 'update Title' WHERE id = 1;
        ```
    ### DELETE (테이블 레코드 삭제)
        ```SQL
        DELETE FROM table_name [WHERE condition] -- 조건 없으면 모든 레코드 삭제

        -- 예시: articles 테이블 1번 레코드 삭제
        DELETE FROM articles WHERE id = 1;

        -- 예시: articles 테이블에서 작성일이 오래된 순으로 레코드 2개 삭제
        DELETE FROM articles WHERE id IN (
            SELECT id FROM articles
            ORDER BY createAt
            LIMIT 2
        );
        ```

## JOIN
- 테이블을 분리한 후 다른 테이블과 결합하여 출력하는 것이 필요할 때 사용

    ### INNER JOIN
    - 두 테이블에서 값이 일치하는 레코드에 대해서만 결과를 반환
        ```SQL
        SELECT select_list
        FROM table_a -- 메인 테이블
        INNER JOIN table_b -- 조인 테이블
            ON table_b.fk = table_a.pk -- join 조건
            -- 조건은 table_a와 table_b 간의 레코드를 일치시키는 규칙으로 지정
        
        -- 예시: 1번 회원이 작성한 모든 게시글의 제목과 작성자명 조회
        SELECT articles.title, user.name
        FROM articles
        INNER JOIN users
            ON users.id = articles.userId
            WHERE users.id = 1;
        ```
    ### LEFT JOIN
    - 왼쪽 테이블 기준으로 다 보여달라
    - 왼쪽 테이블 모든 레코드는 무조건 출력, 오른쪽 테이블은 연결된 값이 없으면 NULL로 표시
    - 회원 중 아직 주문을 안 한 사람 찾기 등 어떤 데이터가 없는지 확인하고 싶을 때 유용
        ```SQL
        SELECT select_list
        FROM table_a -- 메인 테이블: 왼쪽 테이블
        LEFT JOIN table_b --조인 테이블: 오른쪽 테이블
            ON table_b.fk = table_a.pk;
        
        -- 예시: 게시글을 작성한 이력이 없는 회원 정보 조회
        SELECT user.name
        FROM user
        LEFT JOIN articles
            ON articles.userId = users.id
            WHERE articles.userId is NULL;
        ```
