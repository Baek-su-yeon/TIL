## 1번

아래 자바 코드에서 출력되는 값을 작성하시오.

```java
class Static {
    public int a = 20;
    static int b = 0;
}

public class Main {
    public static void main(String[] args) {
        int a;
        a = 10;
        Static.b = a;
        Static st = new Static();
        System.out.println(Static.b++);
        System.out.println(st.b);
        System.out.println(a);
        System.out.print(st.a);
    }
}
```

**정답:** 10 / 11 / 10 / 20

**해설:**
이 문제는 자바의 **static(정적) 변수**와 **인스턴스 변수**의 차이를 묻는 문제입니다.

- `static int b`: 클래스에 딱 하나만 존재하는 변수입니다. 객체를 여러 개 만들어도 `b`는 모두가 공유하는 하나의 값입니다. `Static.b`처럼 클래스 이름으로 접근합니다.
- `public int a`: 객체(인스턴스)를 만들어야 생기는 변수입니다. 객체마다 따로 가집니다.

코드 흐름을 따라가 봅시다.
1. `a = 10;` → 지역변수 a는 10
2. `Static.b = a;` → static 변수 b에 10 저장
3. `System.out.println(Static.b++);` → **후위 증가(b++)**는 "현재 값을 먼저 사용하고, 그 다음에 1을 증가"시킵니다. 따라서 화면에는 **10**이 찍히고, 그 후 b는 11이 됩니다.
4. `System.out.println(st.b);` → b는 이제 11이므로 **11** 출력. (static 변수는 객체 `st`로 접근해도 같은 값을 봅니다.)
5. `System.out.println(a);` → 지역변수 a는 그대로 **10**
6. `System.out.print(st.a);` → 객체의 인스턴스 변수 a는 초기값 **20**

핵심: `b++`는 "출력 후 증가"라는 점, static 변수는 공유된다는 점입니다.

## 2번

다음 C언어의 출력값을 작성하시오.

```c
#include <stdio.h>

int main(){
    char a[] = "Art";
    char* p = NULL;
    p = a;

    printf("%s\n", a);
    printf("%c\n", *p);
    printf("%c\n", *a);
    printf("%s\n", p);

    for(int i = 0; a[i] != '\0'; i++)
        printf("%c", a[i]);
}
```

**정답:** Art / A / A / Art / Art

**해설:**
C언어에서 **문자열은 char 배열**로 저장되고, 배열 이름(`a`)은 그 배열의 **첫 글자 주소**를 의미합니다.

- `char a[] = "Art";` → 메모리에 `A`, `r`, `t`, `\0`(문자열 끝 표시) 가 저장됩니다.
- `p = a;` → 포인터 p가 배열 a의 시작 주소를 가리킵니다. 즉 p와 a는 같은 곳을 가리킵니다.

출력 분석:
- `printf("%s\n", a);` → `%s`는 "주소부터 `\0`까지 문자열 전체"를 출력 → **Art**
- `printf("%c\n", *p);` → `%c`는 한 글자, `*p`는 p가 가리키는 첫 글자 → **A**
- `printf("%c\n", *a);` → `*a`도 배열 첫 글자 → **A**
- `printf("%s\n", p);` → p가 가리키는 문자열 전체 → **Art**
- 마지막 for문: `\0`을 만날 때까지 한 글자씩 출력 → **Art**

핵심: `%s`는 문자열 전체, `%c`는 한 글자. 포인터 앞의 `*`는 "그 주소에 든 값"을 뜻합니다.

## 3번

다음 C언어의 출력값을 작성하시오.

```c
#include <stdio.h>

int main(){
    char* a = "qwer";
    char* b = "qwtety";

    for(int i = 0; a[i] != '\0'; i++){
        for(int j = 0; b[j] != '\0'; j++){
            if(a[i] == b[j]) printf("%c", a[i]);
        }
    }
}
```

**정답:** qwe

**해설:**
**이중 for문**으로 두 문자열을 비교하는 문제입니다. 바깥쪽 for는 a의 각 글자, 안쪽 for는 b의 각 글자를 돕니다. a의 한 글자가 b 안 어딘가에 있으면 그 글자를 출력합니다.

- a = "qwer" → q, w, e, r
- b = "qwtety" → q, w, t, e, t, y

하나씩 확인:
- `q` → b에 q 있음 → 출력 **q**
- `w` → b에 w 있음 → 출력 **w**
- `e` → b에 e 있음 → 출력 **e**
- `r` → b에 r 없음 → 출력 안 함

결과는 **qwe**.

핵심: 안쪽 for문이 b를 처음부터 끝까지 훑으면서 같은 글자가 있는지 검사한다는 흐름을 이해하면 됩니다.

## 5번

아래 내용을 확인하여 괄호 안에 용어를 표 안에 알맞는 값을 고르시오. (보기 예: 패킷 등)

| 구분 | 내용 |
| --- | --- |
| ( ) | 회선교환과 데이터그램 방식의 장점을 결합. 처음 패킷으로 최적 경로를 고정하고, 고정되면 그 다음은 패킷으로 나누어 고속 전송. ATM이 대표적이며 다량 데이터 연속 전송에 적합. |
| ( ) | 패킷교환 방식으로 동작하면서 IP 주소를 사용. 각 패킷을 정해진 경로 없이 독립적으로 처리·교환. 경로를 피해 전송 가능해 신뢰성 높음. 짧은 메시지 전송에 효과적이고 재정렬 필요. |

**정답:** 가상회선, 데이터그램

**해설:**
**패킷 교환 방식**에는 두 종류가 있습니다.

- **가상회선(Virtual Circuit) 방식**: 데이터를 보내기 전에 미리 "경로(길)"를 하나 정해 놓고, 그 길로만 모든 패킷을 순서대로 보냅니다. 연결을 미리 설정하므로 **연결 지향형**이라고 합니다. 순서가 보장되어 재정렬이 필요 없습니다. 대표 예가 **ATM**입니다.
- **데이터그램(Datagram) 방식**: 경로를 미리 정하지 않고, 패킷마다 그때그때 가장 좋은 길로 따로따로 보냅니다. **비연결형**이라 빠르고 유연하지만, 패킷이 도착 순서가 뒤죽박죽이 될 수 있어 **재정렬(순서 맞추기)**이 필요합니다. **IP**가 대표적입니다.

암기 팁: "**가상회선 = 미리 길 고정 = 순서 보장**", "**데이터그램 = 길 제각각 = 재정렬 필요**".

## 6번

아래 내용을 확인하여 알맞는 답을 작성하시오.

- 2계층(데이터링크 계층)에서 구현되는 터널링 기술 중 하나
- L2F와 PPTP가 결합된 프로토콜로 VPN과 ISP가 이용
- IPsec과 함께 사용하면 PPTP보다 안전하지만 보안보다 익명화에 더 적합

**정답:** L2TP

**해설:**
**L2TP**는 "Layer 2 Tunneling Protocol"의 약자로, 이름 그대로 **2계층(데이터링크 계층)**에서 동작하는 **터널링** 프로토콜입니다.

- **터널링(Tunneling)**이란? 공용 인터넷망 위에 마치 비밀 통로(터널)를 뚫어 안전하게 데이터를 보내는 기술입니다. VPN의 핵심 기술입니다.
- L2TP는 두 기존 기술인 **L2F**와 **PPTP**를 합쳐 만든 것입니다.
- L2TP 자체에는 암호화 기능이 약해서, 보통 **IPsec과 함께** 써서 보안을 강화합니다.

연관 암기: VPN 터널링 프로토콜 종류 → **PPTP, L2F, L2TP, IPsec, SSL** 등.

## 9번

다음 아래 코드에서 이진수를 십진수로 변환하는 코드에 대해 괄호 (a)(b)의 적합한 답을 작성하시오.

```c
#include <stdio.h>

int main() {
    int input = 101110;
    int di = 1;
    int sum = 0;

    while (1) {
        if (input == 0) break;
        else {
            sum = sum + (input (a)(b)) * di;
            di = di * 2;
            input = input / 10;
        }
    }

    printf("%d", sum);
    return 0;
}
```

**정답:**
- 1번 풀이: (a) % , (b) 10 (또는 5, 2)
- 2번 풀이: (a) & , (b) 1

**해설:**
이 코드는 **2진수를 10진수로 변환**합니다. 여기서 `input=101110`은 실제로는 십진수처럼 저장된 숫자지만, 각 자리(0,1,1,1,0,1)를 2진수의 자릿수로 취급해 변환합니다.

원리: 맨 끝 자리부터 하나씩 떼어내서 → 2의 거듭제곱(`di`: 1, 2, 4, 8...)을 곱해 더합니다.

- `input % 10` → 맨 끝 한 자리 숫자를 떼어냅니다. (예: 101110 % 10 = 0)
- `input / 10` → 맨 끝 자리를 버립니다. (예: 101110 / 10 = 10111)
- `di = di * 2` → 자릿값을 2배씩 키웁니다.

따라서 **(a)는 `%`, (b)는 `10`**이면 "끝자리 추출" 방식입니다.

또 다른 풀이로 `input & 1`을 쓰면, **비트 AND 연산**으로 맨 끝 비트만 추출할 수 있어 같은 효과가 납니다. (단 이 경우 `input`이 진짜 2진 비트로 처리될 때 유효)

핵심: `% 10` = 끝자리 뽑기, `/ 10` = 끝자리 버리기. 이 두 연산은 자릿수 처리의 기본 패턴입니다.

## 10번

다음 보안 관련 설명으로 괄호 안에 알맞는 용어를 작성하시오.

( )은/는 TCP/IP에서 IP 패킷을 처리할 때 발생되는 문제를 알려주는 프로토콜이다. 다른 호스트나 게이트웨이와 연결된 네트워크에 문제가 있는지 확인하기 위해 사용된다. ( )을/를 이용한 공격에는 ( ) Flooding이 있는데, ping 명령어로 ( ) 패킷을 연속으로 보내 서버가 다른 작업을 못하게 하는 공격이다.

**정답:** ICMP

**해설:**
**ICMP**는 "Internet Control Message Protocol"의 약자로, 네트워크에 **문제가 생겼는지 알려주는** 메시지 전달용 프로토콜입니다.

- 우리가 인터넷 연결을 확인할 때 쓰는 **ping** 명령어가 바로 ICMP를 사용합니다. ping을 보내면 상대가 살아있는지 응답이 옵니다.
- **ICMP Flooding(=Ping of Death, Smurf 등)**: ping 패킷을 어마어마하게 많이 보내서 서버를 마비시키는 **DoS(서비스 거부) 공격**입니다.

핵심: "ping = ICMP", "문제 상황을 알려주는 프로토콜"이라는 키워드를 기억하세요.

## 11번

다음은 디자인 패턴에 관한 설명이다. 알맞는 답을 고르시오.

- 다른 무언가와 이어지는 인터페이스 역할을 하는 클래스
- 실제 객체를 호출하면 행위를 중간에 가로채서 다른 동작을 수행하는 객체로 변경

**정답:** Proxy

**해설:**
**프록시(Proxy) 패턴**은 GoF 디자인 패턴 중 **구조 패턴**에 속합니다. "Proxy"는 "대리인"이라는 뜻입니다.

- 실제 객체에 직접 접근하지 않고, **대리 객체(Proxy)를 거쳐서** 접근하게 만듭니다.
- 이 중간 대리인이 접근을 통제하거나, 무거운 객체의 생성을 미루거나(지연 로딩), 추가 작업을 끼워 넣을 수 있습니다.
- 일상 비유: 연예인에게 직접 연락하지 않고 **매니저(대리인)**를 통하는 것과 같습니다.

암기 팁: "**Proxy = 대리인 = 중간에서 접근 제어**". (참고: 디자인 패턴은 생성/구조/행위 3분류로 나뉘며, Proxy는 구조 패턴입니다.)

다음 코드에서 괄호 안에 알맞는 값을 변수명으로 작성하시오. (출력값: 1 2 3 5 7 8)

```java
public class Sort {
    public static void swap(int[] arr, int idx1, int idx2){
        int temp = arr[idx1];
        arr[idx1] = arr[idx2];
        arr[( 1 )] = temp;
    }

    public static void Usort(int[] array, int length){
        for (int i = 0; i < length; i++) {
            for (int j = 0; j < length - i - 1; j++) {
                if (array[j] > array[j + 1]) {
                    swap(array, j, j + 1);
                }
            }
        }
    }

    public static void main(String[] args) {
        int[] item = new int[] { 5, 3, 8, 1, 2, 7 };
        int nx = 6;
        Usort(item, ( 2 ));
        for (int data : item) {
            System.out.print(data + " ");
        }
    }
}
```

**정답:** (1) idx2 / (2) nx

**해설:**
이 코드는 **버블 정렬(Bubble Sort)**을 구현한 것입니다. 옆에 있는 두 값을 비교해서 큰 값을 뒤로 보내는 방식입니다.

- **(1) 자리: swap 함수**는 두 값을 맞바꾸는 함수입니다.
  ```
  temp = arr[idx1];      // idx1 값을 임시 보관
  arr[idx1] = arr[idx2]; // idx1 자리에 idx2 값을 넣음
  arr[(?)] = temp;       // idx2 자리에 보관해둔 값을 넣어야 교환 완성
  ```
  따라서 빈칸은 **idx2**입니다. (두 값을 서로 바꾸려면 idx2 자리에 temp를 넣어야 함)

- **(2) 자리:** `Usort(item, ?)`에서 두 번째 인자는 배열의 길이(length)입니다. 배열 item의 원소가 6개이고 변수 `nx = 6`이므로 빈칸은 **nx**입니다.

핵심: swap(값 교환)의 3단계 패턴 — "임시 저장 → 한쪽 덮어쓰기 → 임시값으로 나머지 채우기"를 이해하면 됩니다.

## 20번

다음 자바 코드에 대한 출력값을 작성하시오.

```java
class Parent {
    int x = 100;
    Parent() {
        this(500);
    }
    Parent(int x) {
        this.x = x;
    }
    int getX() {
        return x;
    }
}

class Child extends Parent {
    int x = 4000;
    Child() {
        this(5000);
    }
    Child(int x) {
        this.x = x;
    }
}

public class Main {
    public static void main(String[] args) {
        Child obj = new Child();
        System.out.println(obj.getX());
    }
}
```

**정답:** 500

**해설:**
이 문제는 **생성자 호출 흐름**과 **변수 가리기(필드 은닉)**, **메서드 상속**을 동시에 묻습니다. 차근차근 봅시다.

먼저 알아야 할 개념:
- `this(...)`: **같은 클래스의 다른 생성자**를 호출합니다.
- 부모와 자식에 **같은 이름 변수 x**가 있으면 둘은 **별개의 변수**입니다(필드는 오버라이딩 안 됨). Parent의 x, Child의 x가 따로 존재합니다.
- `getX()`는 Parent에만 정의되어 있고, 그 안의 `x`는 **Parent의 x**를 가리킵니다.

흐름:
1. `new Child()` → Child() 생성자 실행 → `this(5000)`로 Child(int x) 호출
2. Child(int x): 그런데 자식 생성자는 실행되기 전에 **부모 생성자를 먼저** 호출합니다(묵시적 super()). → Parent() 호출
3. Parent() → `this(500)` → Parent(int x) 호출 → **Parent의 x = 500**
4. 다시 Child(int x)로 돌아와 `this.x = 5000` → 이건 **Child의 x = 5000** (Parent의 x와 별개!)

마지막에 `obj.getX()`는 Parent에 정의된 메서드이고, 그 안의 x는 **Parent의 x(=500)**를 반환합니다. → **500**

핵심: getX()가 보는 x는 "Parent의 x"입니다. Child의 x(5000)는 영향을 주지 못합니다. **필드는 타입(선언) 기준, 메서드는 객체(실제) 기준**이라는 점을 기억하세요.
