## JS 변수 작성 규칙
- 변수명은 반드시 문자, '$', '_'로 시작
- 특히, '$', '_'로 시작하는 변수는 특별한 용도(DOM 선택, 내부용 변수)로 쓰이므로 일반 변수와 구분 필요

|Naming Convention|사용처|
|:---|:---|
|카멜 케이스(camelCase)|변수, 객체, 함수|
|파스칼 케이스(PascalCase)|클래스, 생성자|
|대문자 스네이크 케이스(SNAKE_CASE)|상수(constants)|

## 변수 선언 키워드
|키워드|설명|
|:---|:---|
|let|**재할당이 필요**한 **변수**를 선언할 때 사용|
|const|**재할당이 불가능**한 **상수**를 선언할 때 사용|
|var|**재선언/재할당이 가능**하나, 호이스팅(Hoisting)문제 고려 필요|

> 호이스팅(hoisting) : 변수 선언문이 코드의 최상단으로 끌어올려지는 듯한 현상
>
> var로 선언된 변수는 선언 위치와 관계 엇이 스코프 최상단에서 선언된 것처럼 동작하므로 고려가 필요하다는 의미 (선언 전에도 사용이 가능하며, 할당 전까지는 undefined 값을 가짐)

### let
- **재할당 가능**, 재선언 불가능
- 블록 스코프를 갖는 **지역 변수**를 선언
    ```javascript
    let number = 10 // 선언 및 초기값 할당
    number = 20 // 재할당 가능
    let number = 20 // 재선언 불가능
    ```
### const
- **재할당 불가능**, 재선언 불가능
- 블록 스코프를 갖는 **지역 변수**를 선언
    ```javascript
    const number = 10 // 선언 및 초기값 할당
    number = 10 // 재할당 불가능
    const number = 20 // 재선언 불가능
    ```

> 🚨 코드 의도 명확화, 버그 예방을 위해 **const를 기본 사용**하고 재할당이 필요한 경우에 let으로 전환

## DOM (Document Object Model)
- 웹 페이지(Document)를 구조화된 객체로 제공하여 프로그래밍 언어가 페이지 구조에 접근할 수 있는 방법을 제공
- head, body, h1, p, a 등 HTML의 각 태그가 나타내는 구역(상자)를 각 객체로 볼 수 있음
- 웹페이지를 부모-자식 관계의 계층적인 트리 구조로 표현 (즉, 객체 간 상속 구조가 존재)

    ### document 객체
    - 웹 페이지를 나타내는 DOM 트리의 **최상위 객체**
    - HTML 문서의 모든 콘텐츠에 접근하고 조작할 수 있는 진입점

    ### DOM 선택
    |선택 메서드|설명|예시|
    |:---|:---|:---|
    |querySelector|요소 **한 개** 선택, 제공한 선택자를 만족하는 **첫 번째 element 객체**를 반환 없으면 null 반환|document.querySelector(selector)|
    |querySelectorAll|요소 **여러 개** 선택, 제공한 선택자를 만족하는 **NodeList**를 반환|document.querySelectorAll(selector)|

    > selector: 찾고 싶은 html 요소를 지정하는 검색 조건 (예: 클래스명, 아이디명 등)

    ### 속성 조작
    1. 클래스 속성 조작 ('classList' property)
    
        스타일링 및 상태 제어를 위한 클래스 목록을 동적으로 추가/제거

        요소의 클래스 목록을 **DOMTokenList(유사 배열) 형태**로 반환

        |메서드|설명|예시|
        |:---|:---|:---|
        |add|지정한 클래스 값을 추가|element.classList.add()|
        |remove|지정한 클래스 값을 제거|element.classList.remove()|
        |toggle|클래스가 존재한다면 제거하고 false 반환, 존재하지 않으면 추가하고 true를 반환|element.classList.toggle()|        

    2. 일반 속성 조작 (Attribute)

        id, href 등 요소의 모든 HTML 속성 값을 직접 설정/조회

        |메서드|설명|예시|
        |:---|:---|:---|
        |getAttribute()|해당 요소에 지정된 값을 반환(조회)|element.getAttribute()|
        |setAttribute|지정된 요소의 속성 값을 설정, 속성이 이미 있으면 기존 값을 갱신|element.setAttribute(name,value)|
        |removeAttribute()|요소에서 지정된 이름을 가진 속성 제거|element.removeAttribute()|

        > 🚨 setAttribute()는 숫자나 boolean 값을 넣어도 모두 **문자열로 반환하여 저장**

    ### HTML 콘텐츠 조작 ('textContent' property)

    - HTML 태그를 완전히 제거하고 순수한 텍스트 데이터만 얻고 싶을 때 유용

        ```javascript
        const h1Tag = document.querySelector('.heading')
        console.log(h1Tag.textContent) // console 창에 현재 h1Tag 텍스트 데이터 출력

        h1Tag.textContent = '내용 수정'
        console.log(h1Tag.textContent) // console 창에 내용 수정 출력
        ```

    ### DOM 요소 조작

    |메서드|설명|예시|
    |:---|:---|:---|
    |createElement|작성한 tagName의 HTML 요소를 생성하여 반환|document.createElement(tagName)|
    |appendChild|한 Node를 특정 부모 Node의 자식 NodeList 중 **마지막 자식으로 삽입**|Node.appendchild()|
    |removeChild|DOM에서 자식 Node를 제거하고 제거된 Node를 반환|Node.removeChild()|

    > 🚨 'createElement'는 메모리에만 요소를 만들고, 실제로는 'appendChild'로 문서에 삽입해야 화면에 표출
        
    > ✳ 'removeChild'로 제거된 노드는 메모리에 남아있으며, 변수에 담아두면 언제든 사용 가능

    ### Style 조작 ('style' property)

    - 해당 요소의 모든 style 속성 목록을 포함하는 속성
    - 즉, HTML 태그에 style = "..." 속성을 직접 추가/수정하는 것과 동일
    - **하지만 보통 스타일 변경은 클래스를 활용**

## 용어 정리

1. **Node**
    - DOM의 기본 구성 단위
    - Document Node = HTML 문서 전체를 나타내는 노드
    - Element Node = HTML 요소를 나타내는 노드 (예: p 태그)

2. **NodeList**
    - DOM 메서드를 사용해 선택한 Node의 목록
    - 배열과 유사한 구조를 가지며, index로만 각 항목에 접근 가능
    - JavaScript의 배열 메서드 사용 가능
    - querySelectorAll()에 의해 반환되는 NodeList는 DOM의 변경사항을 실시간으로 반영 ❌

3. **Element**
    - DOM 트리에서 HTML 요소를 나타내는 특별한 유형의 Node
    - Node의 하위 유형이며, p 태그, div 태그, span 태그, body 태그 등 HTML 태그들이 Element 노드를 생성
    - Node의 속성과 메서드를 모두 가지고 있으며 추가적으로 요소 특화된 기능(className, innerHTML, id 등)을 가짐
