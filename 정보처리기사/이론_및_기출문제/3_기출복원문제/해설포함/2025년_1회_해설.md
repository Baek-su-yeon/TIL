## 1번

다음은 네트워크 보안에 관련된 문제이다. 괄호 안에 알맞는 용어를 작성하시오.

( )은/는 '세션을 가로채다.'라는 의미로 다른 사람의 세션 상태를 훔치거나 도용하여 액세스하는 해킹 기법이다. TCP ( )은/는 3-way 핸드셰이크가 완료된 후 공격자가 시퀀스 번호 등을 조작하여 정상 세션을 가로채고 인증 없이 통신을 탈취하는 공격이다.

**정답:** 세션 하이재킹

**해설:**
**세션 하이재킹(Session Hijacking)**은 이미 인증을 마친 사용자의 **세션(연결 상태)을 가로채서** 그 사람인 척 통신을 탈취하는 공격입니다.

- '세션'이란 로그인 후 서버가 "이 사용자는 인증됨"이라고 기억하는 연결 상태입니다.
- TCP는 연결할 때 **3-way 핸드셰이크**(SYN → SYN/ACK → ACK)로 시작하고, 각 패킷에 **시퀀스 번호**를 붙여 순서를 관리합니다.
- 공격자는 이 시퀀스 번호를 알아내 조작함으로써, 정상 사용자가 인증한 세션에 끼어들어 **별도 인증 없이** 통신을 빼앗습니다.

키워드 "세션 탈취 + 시퀀스 번호 조작 = 세션 하이재킹"으로 기억하세요.

아래 내용에서 설명 글의 괄호 안의 용어를 영문 약자로 작성하시오.

( )은/는 3글자의 영어 약자로 이루어진 오류 검출 코드로, 데이터에 체크섬을 추가하여 전송/저장 후 수신·읽기 시 다시 계산하여 변경 여부를 확인한다. 정해진 다항식(x³ + x + 1)을 기반으로 2진수 나눗셈을 하고 나머지를 ( ) 값으로 삼는다.

**정답:** CRC

**해설:**
**CRC(Cyclic Redundancy Check, 순환 중복 검사)**는 데이터 전송 오류를 검출하는 대표적인 방법입니다.

- 송신 측에서 데이터를 **정해진 다항식으로 나눗셈(2진수 나눗셈)**한 뒤 그 **나머지를 검사값(CRC)**으로 만들어 데이터 뒤에 붙여 보냅니다.
- 수신 측에서 같은 방식으로 다시 계산해, 나머지가 0이 아니면 "전송 중 데이터가 변했다"고 판단합니다.
- 단순 패리티 검사보다 더 강력하게 오류를 잡아낼 수 있습니다.

키워드 "다항식 나눗셈 + 나머지 = 검사값 = CRC"입니다. 오류 검출(탐지)만 하고 정정은 하지 않는다는 점도 알아두세요. (오류 정정까지 하는 것은 Hamming 코드입니다.)

## 5번

다음은 Java 코드에 대한 문제이다. 알맞는 출력값을 작성하시오.

```java
public class Main {
    public static void main(String[] args) {
        int a=5, b=0;
        try {
            System.out.print(a/b);
        } catch(ArithmeticException e) {
            System.out.print("출력1");
        } catch(ArrayIndexOutOfBoundsException e) {
            System.out.print("출력2");
        } catch(NumberFormatException e) {
            System.out.print("출력3");
        } catch(Exception e) {
            System.out.print("출력4");
        } finally {
            System.out.print("출력5");
        }
    }
}
```

**정답:** 출력1출력5

**해설:**
자바의 예외 처리와 **0으로 나누기 예외**를 다루는 문제입니다.

- `a/b`에서 b=0이므로 **0으로 나누기**가 일어납니다. 자바에서 정수를 0으로 나누면 `ArithmeticException`이 발생합니다.
- catch 절을 위에서부터 확인 → **첫 번째 `catch(ArithmeticException e)`가 일치** → "출력1" 출력.
- 한 번 잡으면 나머지 catch들은 모두 건너뜁니다.
- `finally`는 항상 실행 → "출력5" 출력.

따라서 "출력1출력5". `System.out.print(a/b)`에서 나눗셈이 먼저 예외를 일으키므로 print는 아무것도 출력하지 못하고 곧장 catch로 넘어갑니다. 예외 종류(0으로 나누기 = ArithmeticException)를 알아두세요.

## 6번

아래 내용은 ARP/RARP에 대한 설명이다. 각 설명에 해당하는 것을 작성하시오.

( 1 )은/는 네트워크상에서 IP 주소를 MAC 주소로 변환하는 프로토콜이고, ( 2 )은/는 MAC 주소를 IP 주소로 변환하는 프로토콜이다.

**정답:** (1) ARP / (2) RARP

**해설:**
**ARP**와 **RARP**는 IP 주소와 MAC 주소를 서로 변환하는 프로토콜입니다.

- IP 주소: 논리적 주소(인터넷에서 사용하는 주소).
- MAC 주소: 물리적 주소(네트워크 카드에 새겨진 고유 번호).

- **ARP(Address Resolution Protocol)**: **IP 주소 → MAC 주소**로 변환. "이 IP를 가진 기기의 물리 주소가 뭐지?"를 알아냅니다.
- **RARP(Reverse ARP)**: 반대로 **MAC 주소 → IP 주소**로 변환. 'Reverse(거꾸로)'라는 이름 그대로입니다.

암기 팁: "ARP는 IP를 MAC으로(보통 방향), RARP는 그 반대(Reverse)". 둘의 방향만 헷갈리지 않으면 됩니다.

## 7번

다음은 SQL 문제이다. 아래 두 테이블을 참고하여 쿼리 실행 결과를 작성하시오. (※ 원문에 테이블 이미지 포함)

```sql
SELECT name, incentive FROM emp, sal
WHERE emp.id = sal.id and incentives >= 500
```

**정답:**
| name | incentives |
| --- | --- |
| 이순신 | 1000 |

**해설:**
두 테이블을 **조인(JOIN)**하여 조건에 맞는 행을 뽑는 문제입니다.

- `FROM emp, sal WHERE emp.id = sal.id` → 이것은 두 테이블을 **공통 컬럼(id)으로 연결**하는 방식입니다. (쉼표로 나열하고 WHERE로 연결 조건을 거는 옛 방식의 조인)
- `and incentives >= 500` → 거기에 추가로 "인센티브가 500 이상인" 행만 남깁니다.

두 조건을 모두 만족하는 행을 찾으면, id가 일치하면서 인센티브가 500 이상인 사람은 "이순신(1000)"뿐입니다.

핵심: 조인은 두 테이블을 공통 키로 묶는 것이고, WHERE에 조인 조건과 일반 조건을 함께 쓸 수 있습니다.

1. 릴레이션에서 속성의 개수를 의미 : ( 1 )
2. 릴레이션에서 튜플의 개수를 의미 : ( 2 )
3. 한 릴레이션의 속성이 다른 릴레이션의 기본 키를 참조할 때, 참조하는 속성 : ( 3 )

**[보기]** ㄱ. domain ㄴ. primary ㄷ. degree ㄹ. candidate ㅁ. cardinality ㅂ. attribute ㅅ. foreign

- degree = 열 개수 ↔ cardinality = 행 개수

## 9번

IP 주소가 192.168.35.10, 서브넷 255.255.252.0인 PC에서 브로드캐스팅으로 다른 IP로 정보를 전달할 때 수신할 수 있는 IP를 보기에서 골라 모두 작성하시오.

**[보기]** ㄱ. 192.168.34.1 ㄴ. 192.168.32.19 ㄷ. 192.168.35.200 ㄹ. 192.168.33.138 ㅁ. 192.168.35.50

**정답:** ㄱ, ㄴ, ㄷ, ㄹ, ㅁ

**해설:**
**브로드캐스트**는 "같은 네트워크(서브넷) 안의 모든 기기"에게 한꺼번에 보내는 것입니다. 따라서 **같은 네트워크에 속한 IP들만** 수신할 수 있습니다. 핵심은 "어디까지가 같은 네트워크인가"를 서브넷 마스크로 계산하는 것입니다.

서브넷 마스크 255.255.252.0을 분석합니다.
- 세 번째 옥텟이 252 = 2진수 `11111100`입니다. 즉 상위 6비트가 네트워크, 하위 2비트가 호스트용입니다.
- 한 네트워크가 차지하는 세 번째 옥텟의 범위는 **4씩 묶입니다** (256 − 252 = 4).
- 32, 33, 34, 35가 하나의 묶음(블록)입니다. (32부터 4개)

내 IP의 세 번째 옥텟은 35 → 속한 블록은 **세 번째 옥텟이 32~35인 네트워크**입니다. 즉 192.168.32.0 ~ 192.168.35.255가 같은 네트워크입니다.

보기를 확인하면:
- ㄱ 192.168.34.1, ㄴ 192.168.32.19, ㄷ 192.168.35.200, ㄹ 192.168.33.138, ㅁ 192.168.35.50

세 번째 옥텟이 모두 32~35 범위 안 → **전부 같은 네트워크** → 모두 수신 가능(ㄱ,ㄴ,ㄷ,ㄹ,ㅁ).

핵심: 서브넷 마스크로 네트워크 블록 크기를 구하고(여기선 4 단위), 같은 블록에 속하는지 확인합니다.

## 10번

다음은 C언어에 대한 문제이다. 알맞는 출력값을 작성하시오.

```c
#include <stdio.h>
char Data[5] = {'B', 'A', 'D', 'E'};
char c;

int main(){
    int i, temp, temp2;
    c = 'C';
    printf("%d\n", Data[3]-Data[1]);

    for(i=0; i<5; ++i){
        if(Data[i] > c)
            break;
    }

    temp = Data[i];
    Data[i] = c;
    i++;

    for(; i<5; ++i){
        temp2 = Data[i];
        Data[i] = temp;
        temp = temp2;
    }

    for(i=0; i<5; i++){
        printf("%c", Data[i]);
    }
}
```

**정답:** 4 / BACDE

**해설:**
**문자(char)는 내부적으로 아스키 코드 숫자**로 저장된다는 점이 핵심입니다. (A=65, B=66, C=67, D=68, E=69, …)

먼저 첫 출력: `Data[3] - Data[1]` = 'E' − 'A' = 69 − 65 = **4**.

배열 초기 상태: Data = {'B', 'A', 'D', 'E', '\0'(빈 자리는 0)}

c = 'C'(67). 첫 for문은 **c보다 큰 첫 문자**를 찾습니다.
- Data[0]='B'(66) > 'C'(67)? 아니오
- Data[1]='A'(65) > 'C'? 아니오
- Data[2]='D'(68) > 'C'? **예** → i=2에서 break

이제 'C'를 i=2 자리에 끼워 넣고, 원래 있던 값들을 한 칸씩 뒤로 밉니다(삽입 정렬과 비슷).
- temp = Data[2]='D' 보관, Data[2]='C'로 교체, i를 3으로.
- i=3: temp2=Data[3]='E' 보관, Data[3]=temp('D'), temp='E'.
- i=4: temp2=Data[4]=빈값, Data[4]=temp('E'), …

결과 배열: B, A, C, D, E → 마지막 for문이 출력 → **"BACDE"**

종합 출력: "4"(줄바꿈) 다음 "BACDE". 'C'를 알맞은 위치에 끼워 넣어 정렬된 모양을 만드는 과정입니다.

## 11번

다음은 C언어에 대한 문제이다. 알맞는 출력값을 작성하시오.

```c
#include <stdio.h>
#include <stdlib.h>

void set(int** arr, int* data, int rows, int cols) {
    for (int i = 0; i < rows * cols; ++i) {
        arr[((i + 1) / rows) % rows][(i + 1) % cols] = data[i];
    }
}

int main() {
    int rows = 3, cols = 3, sum = 0;
    int data[] = {5, 2, 7, 4, 1, 8, 3, 6, 9};
    int** arr;
    arr = (int**) malloc(sizeof(int*) * rows);

    for (int i = 0; i < cols; i++) {
        arr[i] = (int*) malloc(sizeof(int) * cols);
    }

    set(arr, data, rows, cols);

    for (int i = 0; i < rows * cols; i++) {
        sum += arr[i / rows][i % cols] * (i % 2 == 0 ? 1 : -1);
    }

    for(int i=0; i<rows; i++) {
        free(arr[i]);
    }
    free(arr);

    printf("%d", sum);
}
```

**정답:** 13

**해설:**

```python
class Node:
    def __init__(self, value):
        self.value = value
        self.children = []

def tree(li):
    nodes = [Node(i) for i in li]
    for i in range(1, len(li)):
        nodes[(i - 1) // 2].children.append(nodes[i])
    return nodes[0]

def calc(node, level=0):
    if node is None:
        return 0
    return (node.value if level % 2 == 1 else 0) + sum(calc(n, level + 1) for n in node.children)

li = [3, 5, 8, 12, 15, 18, 21]
root = tree(li)
print(calc(root))
```

**정답:** 13

**해설:**
배열을 **이진 트리**로 만든 뒤, **홀수 레벨에 있는 노드의 값만** 더하는 문제입니다.

`tree` 함수는 배열을 완전 이진 트리로 만듭니다. `(i-1)//2`가 부모 인덱스이므로:
```
        index0(3)          ← level 0
       /         \
   index1(5)   index2(8)   ← level 1
   /    \       /    \
 idx3  idx4   idx5  idx6
 (12)  (15)   (18)  (21)   ← level 2
```

`calc`는 `level % 2 == 1`(홀수 레벨)일 때만 값을 더합니다.
- level 0 (값 3): 짝수 레벨 → 더하지 않음 (0)
- level 1 (값 5, 8): 홀수 레벨 → **5 + 8 = 13** 더함
- level 2 (값 12, 15, 18, 21): 짝수 레벨 → 더하지 않음

따라서 합 = **13**.

핵심: 배열 인덱스로 부모/자식 관계를 만드는 공식(부모 = (i-1)//2)과, 레벨의 짝/홀 판별(level % 2)을 이해하면 됩니다.

```c
    new_node->value = value;

Data* reconnect(Data* head, int value) {
    if (head == NULL || head->value == value) return head;
    Data *prev = NULL, *curr = head;
    while (curr != NULL && curr->value != value) {
        prev = curr;
        curr = curr->next;

        curr->next = head;
        head = curr;
    }
    return head;
}

    for (int i = 1; i <= 5; i++)
        head = insert(head, i);
    head = reconnect(head, 3);

        printf("%d", curr->value);
    return 0;
}
```

```java

        System.out.println(calc("5"));

    static int calc(int value) {
        if (value <= 1) return value;
        return calc(value - 1) + calc(value - 2);
```

**해설:**
**오버로딩된 두 개의 calc**가 있습니다. 처음 호출은 문자열 "5"이므로 `calc(String)`이 불립니다. 하지만 그 안에서 `calc(value-1)`처럼 **정수**를 넘기면 `calc(int)`가 호출됩니다. 즉 한 번 int 버전으로 들어가면 계속 int 버전만 씁니다.

- calc(3)=calc(2)+calc(1)=1+1=2

핵심: 첫 호출만 String 버전이고, 그 내부의 재귀는 모두 int 버전(`calc(n-1)+calc(n-2)`)을 탑니다. String 버전의 `value-3` 공식은 처음 한 번만 적용되어 calc(4)+calc(2)로 분기된다는 점을 정확히 봐야 합니다.
