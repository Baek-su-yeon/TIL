# 1. 포인터의 의미

**포인터는 변수의 주소를 저장하는 변수**야.

일반 변수는 값을 저장하고, 포인터 변수는 그 값이 저장된 **메모리 주소**를 저장해.

```c
int a = 50;
int *p = &a;
```

이 코드에서:

```
a  → 값 50을 저장한 일반 변수
&a → a의 주소
p  → a의 주소를 저장한 포인터 변수
*p → p가 가리키는 주소에 있는 값, 즉 a의 값
```

즉 핵심은 이거야.

```
&변수명  → 변수의 주소
*포인터  → 포인터가 가리키는 주소에 있는 값
```

예를 들어:

```c
*p = *p + 20;
```

은 `p`가 `a`를 가리키고 있다면:

```c
a = a + 20;
```

과 같은 효과가 나.

---

# 2. 포인터를 사용하는 이유

포인터를 사용하는 가장 큰 이유는 **값 자체를 복사해서 다루지 않고, 값이 있는 주소를 통해 원본에 접근하기 위해서**야.

대표적인 이유는 3가지야.

## 첫째, 원본 값을 직접 수정하기 위해서

함수에 값을 그대로 넘기면 보통 복사본이 넘어가.

```c
void change(int x) {
    x = 100;
}
```

이 경우 함수 안에서 `x`를 바꿔도 원본 변수는 바뀌지 않아.

하지만 주소를 넘기면:

```c
void change(int *p) {
    *p = 100;
}
```

포인터가 원본 변수의 주소를 가지고 있으므로 원본 값을 직접 바꿀 수 있어.

---

## 둘째, 큰 데이터를 복사하지 않기 위해서

구조체나 배열처럼 크기가 큰 데이터를 함수에 그대로 넘기면 데이터 전체가 복사될 수 있어.

```c
struct Student {
    char name[100];
    int score[50];
};
```

이런 구조체를 그대로 넘기면 부담이 커질 수 있어.

```c
void print(struct Student s)
```

하지만 포인터로 넘기면 구조체 전체가 아니라 주소만 넘기면 돼.

```c
void print(struct Student *p)
```

그래서 메모리와 처리 비용을 줄일 수 있어.

---

## 셋째, 배열이나 구조체 배열을 주소 기반으로 다루기 위해서

배열은 연속된 메모리 공간에 저장돼.

```c
int arr[3] = {10, 20, 30};
```

포인터를 이용하면 주소를 이동하면서 배열 원소에 접근할 수 있어.

```c
int *p = arr;

*p       // arr[0]
*(p + 1) // arr[1]
*(p + 2) // arr[2]
```

구조체 배열도 마찬가지야.

```c
struct jsu st[3];
struct jsu *p = &st[0];
```

그러면:

```
p       → st[0]
p + 1   → st[1]
p + 2   → st[2]
```

처럼 주소를 이동하면서 구조체를 다룰 수 있어.

---

# 3. 포인터를 사용했을 때의 장점

포인터를 사용하면 다음과 같은 장점이 있어.

## 1. 원본 데이터를 직접 수정할 수 있다

포인터는 원본의 주소를 저장하므로, 그 주소를 따라가서 원본 값을 바꿀 수 있어.

```c
int a = 10;
int *p = &a;

*p = 20;
```

결과적으로 `a`는 20이 돼.

---

## 2. 큰 데이터를 효율적으로 다룰 수 있다

데이터 전체를 복사하지 않고 주소만 전달하므로, 구조체나 배열처럼 큰 데이터를 다룰 때 효율적이야.

```
데이터 전체 복사 → 부담 큼
주소만 전달 → 부담 작음
```

---

## 3. 배열과 문자열을 유연하게 다룰 수 있다

문자열도 결국 문자 배열이고, 문자열 포인터는 첫 글자의 주소를 가리켜.

```c
char *s = "gilbut";
```

이때:

```
s       → 첫 문자 'g'의 주소
*s      → 'g'
s[2]    → 'l'
*(s+2)  → 'l'
s+2     → "lbut"의 시작 주소
```

그래서 포인터를 이용하면 문자열의 중간 위치부터 출력하거나, 배열의 특정 위치를 쉽게 다룰 수 있어.

---

## 4. 구조체를 주소로 다룰 수 있다

구조체 포인터를 사용하면 구조체의 주소를 통해 멤버에 접근할 수 있어.

```c
struct jsu *p = &st[0];
```

이때:

```c
p->os
```

는

```c
(*p).os
```

와 같은 뜻이야.

즉:

```
p->os = p가 가리키는 구조체의 os 멤버
```

구조체 포인터를 쓰면 구조체 배열을 이동하거나, 원본 구조체를 직접 수정할 수 있어.

---

```
포인터의 의미
→ 변수의 주소를 저장하는 변수

포인터를 사용하는 이유
→ 값 자체를 복사하지 않고 주소를 통해 원본에 접근하기 위해서

포인터의 장점
→ 원본 수정 가능
→ 큰 데이터 복사 비용 감소
→ 배열, 문자열, 구조체를 효율적으로 처리 가능
```

진짜 시험용으로 한 문장으로 외우면:

```
포인터는 변수의 주소를 저장하는 변수이며, 원본 데이터에 직접 접근하고 큰 데이터의 복사를 줄이기 위해 사용한다.
```
