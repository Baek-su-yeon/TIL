## 1번

다음 Java 코드에서 알맞는 출력값을 작성하시오.

```java
class Connection {
    private static Connection _inst = null;
    private int count = 0;

    static public Connection get() {
        if(_inst == null) {
            _inst = new Connection();
            return _inst;
        }
        return _inst;
    }

    public void count() {
        count++;
    }

    public int getCount() {
        return count;
    }
}

public class main {
    public static void main(String[] args) {
        Connection conn1 = Connection.get();
        conn1.count();
        Connection conn2 = Connection.get();
        conn2.count();
        Connection conn3 = Connection.get();
        conn3.count();
        conn1.count();
        System.out.print(conn1.getCount());
    }
}
```

**정답:** 4

**해설:**
이 코드는 **싱글톤(Singleton) 패턴**을 구현한 것입니다. 싱글톤은 "객체를 단 하나만 만들어 모두가 공유"하는 패턴입니다.

핵심 원리:
- `static Connection _inst`는 클래스에 하나뿐인 변수입니다.
- `get()`은 `_inst`가 비어있으면(null) 새로 만들고, 이미 있으면 **기존 객체를 그대로** 돌려줍니다. → 즉 **항상 같은 객체** 하나만 반환됩니다.

따라서 conn1, conn2, conn3는 **이름만 다를 뿐 전부 같은 객체**를 가리킵니다. count() 호출 횟수를 모두 합산합니다.

- `conn1.count()` → count=1
- `conn2.count()` → (같은 객체) count=2
- `conn3.count()` → count=3
- `conn1.count()` → count=4
- `conn1.getCount()` → **4**

핵심: 변수 3개가 같은 객체를 공유하므로, count는 누적되어 총 **4**번 증가합니다. "객체가 하나뿐"이라는 싱글톤의 핵심을 이해하면 풀립니다.

## 2번

다음 C언어 코드에서 알맞는 출력값을 작성하시오.

```c
#include <stdio.h>

int main() {
    int v1 = 0, v2 = 35, v3 = 29;
    if(v1 > v2 ? v2 : v1) {
        v2 = v2 << 2;
    } else {
        v3 = v3 << 2;
    }
    printf("%d", v2+v3);
}
```

**정답:** 151

**해설:**
**삼항 연산자**와 **비트 시프트 연산**을 묻는 문제입니다.

먼저 if 조건의 삼항 연산자 `v1 > v2 ? v2 : v1`:
- `v1 > v2` → `0 > 35` → 거짓 → 콜론 뒤의 값 `v1`(=0)을 선택합니다.
- if문은 **0이면 거짓**, **0이 아니면 참**으로 봅니다. 결과가 0이므로 **else로 갑니다.**

else 실행: `v3 = v3 << 2;`
- **`<< 2`는 왼쪽으로 2비트 시프트**, 즉 **×4**(2의 2제곱)와 같습니다.
- v3 = 29 × 4 = **116**

마지막: `v2 + v3` = 35 + 116 = **151**

핵심:
- 삼항 연산자 `조건 ? A : B`는 조건이 참이면 A, 거짓이면 B.
- **`<< n`은 ×2ⁿ**, **`>> n`은 ÷2ⁿ**. (시프트는 곱셈/나눗셈의 빠른 버전)

## 3번

다음은 응집도와 관련해서 보기에서 응집도가 높은 순으로 나열하시오.

**보기:** ㄱ. 기능 ㄴ. 교환 ㄷ. 우연 ㄹ. 시간

**정답:** ㄱ, ㄴ, ㄹ, ㄷ

**해설:**
**응집도(Cohesion)**는 모듈 내부 요소들이 **얼마나 서로 관련되어 한 가지 일에 집중하는지**를 나타냅니다. **응집도는 높을수록 좋습니다**(좋은 설계).

응집도 7단계 (높음 → 낮음):
1. **기능적(Functional)** — 가장 좋음. 모든 요소가 하나의 목적에 집중.
2. **순차적(Sequential)** — 한 요소의 출력이 다음 요소의 입력.
3. **교환적/통신적(Communicational)** — 같은 입력/출력 자료를 사용.
4. **절차적(Procedural)** — 정해진 순서로 실행.
5. **시간적(Temporal)** — 같은 시간대에 실행되는 것끼리.
6. **논리적(Logical)** — 비슷한 성격끼리.
7. **우연적(Coincidental)** — 가장 나쁨. 아무 관련 없이 모임.

문제의 보기 중에서 높은 순: **기능(1위) > 교환(3위) > 시간(5위) > 우연(7위)** → **ㄱ, ㄴ, ㄹ, ㄷ**

암기 팁(높→낮): "**기-순-교(통)-절-시-논-우**". 첫 글자만 따서 외우세요. 반대로 **결합도는 낮을수록 좋습니다**(헷갈리지 말 것).

## 4번

다음은 C언어에 대한 문제이다. 알맞는 출력값을 작성하시오.

```c
#include <stdio.h>
#include <string.h>

void reverse(char* str){
    int len = strlen(str);
    char temp;
    char* p1 = str;
    char* p2 = str + len - 1;
    while(p1 < p2){
        temp = *p1;
        *p1 = *p2;
        *p2 = temp;
        p1++;
        p2--;
    }
}

int main(int argc, char* argv[]){
    char str[100] = "ABCDEFGH";
    reverse(str);
    int len = strlen(str);
    for(int i=1; i<len; i+=2){
        printf("%c", str[i]);
    }
    printf("\n");
    return 0;
}
```

**정답:** GECA

**해설:**
두 단계로 나뉩니다: ① 문자열 뒤집기, ② 홀수 인덱스만 출력.

**① reverse 함수**: 양쪽 끝에서 포인터 두 개(p1=앞, p2=뒤)가 서로 마주보며 글자를 바꿔 문자열을 뒤집습니다.
- "ABCDEFGH" → 뒤집으면 **"HGFEDCBA"**

**② 출력 for문**: `i=1`부터 시작해 `i+=2`(2씩 증가), 즉 **홀수 인덱스(1,3,5,7)**만 출력합니다.

뒤집힌 "HGFEDCBA"의 인덱스:
```
H(0) G(1) F(2) E(3) D(4) C(5) B(6) A(7)
```
- 인덱스 1 → G
- 인덱스 3 → E
- 인덱스 5 → C
- 인덱스 7 → A

결과: **GECA**

핵심: "두 포인터가 마주보며 교환 = 문자열 뒤집기"의 전형 패턴, 그리고 `i+=2`는 한 칸씩 건너뛰며 진행한다는 것을 파악하면 됩니다.

## 5번

아래 그림에서의 네트워크에서 라우터를 통한 할당 가능한 2번, 4번, 5번의 IP를 작성하시오. (※ 원문에 네트워크 이미지 포함)

1) 192.168.35.3/24 / 3) 129.200.10.16/22 / 6) 192.168.36.24/24

**보기:** 192.168.35.0, 192.168.35.72, 192.168.36.0, 192.168.36.249, 129.200.8.0, 129.200.8.249

**정답:** 2) 192.168.35.72 / 4) 129.200.8.249 / 5) 192.168.36.249

**해설:**
**서브넷과 호스트 할당** 문제입니다. 핵심은 "**네트워크 주소**와 **브로드캐스트 주소**는 호스트에 줄 수 없다"는 규칙입니다.

기본 개념:
- IP 주소는 **네트워크 부분 + 호스트 부분**으로 나뉩니다. `/24`는 앞 24비트가 네트워크라는 뜻입니다.
- **네트워크 주소**: 호스트 부분이 전부 0 (그 네트워크의 첫 주소). 호스트에 줄 수 없음.
- **브로드캐스트 주소**: 호스트 부분이 전부 1 (마지막 주소). 호스트에 줄 수 없음.
- 그 사이의 주소들만 실제 기기(호스트)에 할당 가능합니다.

보기에서 `.0`으로 끝나는 주소(192.168.35.0, 192.168.36.0, 129.200.8.0)는 네트워크 주소라 제외됩니다. 따라서 할당 가능한 일반 호스트 주소는:
- 2) **192.168.35.72** (같은 35.x 대역, /24)
- 4) **129.200.8.249** (3번의 /22 대역에 속하는 호스트)
- 5) **192.168.36.249** (6번의 36.x 대역, /24)

핵심: "**.0(네트워크), 맨끝(브로드캐스트)는 호스트에 못 줌**"이라는 규칙으로 보기를 거르면 됩니다.

## 6번

아래 표에서 나타나고 있는 정규형을 작성하시오.

| 고객아이디 | 강좌명 | 강사번호 |
| --- | --- | --- |
| apple | 영어회화 | P001 |
| banana | 기초토익 | P002 |
| carrot | 영어회화 | P001 |
| carrot | 기초토익 | P004 |
| orange | 영어회화 | P003 |
| orange | 기초토익 | P004 |

**정답:** 제3정규형

**해설:**
**정규화(Normalization)**는 데이터 중복과 이상현상을 줄이기 위해 테이블을 나누는 과정입니다. 이 표가 어느 단계까지 만족하는지 판단하는 문제입니다.

정규형 단계 요약:
- **1NF(제1정규형)**: 모든 값이 **원자값**(쪼갤 수 없는 하나의 값). 표의 각 칸에 값이 하나씩 → 만족.
- **2NF(제2정규형)**: 1NF + **부분 함수 종속 제거**(기본 키 일부에만 의존하는 속성 제거).
- **3NF(제3정규형)**: 2NF + **이행적 함수 종속 제거**(A→B→C 형태의 간접 종속 제거).

이 표는 (고객아이디, 강좌명)이 기본 키이고, 강사번호가 키 전체에 종속됩니다. 이행적 종속도 없으므로 **제3정규형**을 만족합니다.

암기 팁(정규화 순서): "**1NF(원자값) → 2NF(부분종속 제거) → 3NF(이행종속 제거) → BCNF → 4NF(다치종속) → 5NF(조인종속)**". 두문자: "**원-부-이-결-다-조**".

## 7번

아래의 내용에서 설명하는 네트워크 용어를 영문 약자로 작성하시오.

1. 대표적인 링크 상태 라우팅 프로토콜로, 연결된 링크의 상태를 감시하여 최적의 경로를 선택한다.
2. 단일 자율 시스템 내에서 라우팅 정보를 배포하는 내부 게이트웨이 프로토콜이다.
3. 최단 경로는 Dijkstra 알고리즘을 사용하여 계산된다.

**정답:** OSPF (Open Shortest Path First)

**해설:**
**OSPF(Open Shortest Path First)**는 **링크 상태(Link State)** 방식의 라우팅 프로토콜입니다.

- **링크 상태 방식**: 네트워크 전체의 연결 상태(지도)를 파악한 뒤, **다익스트라(Dijkstra) 알고리즘**으로 최단 경로를 계산합니다.
- **내부 게이트웨이 프로토콜(IGP)**: 하나의 조직(자율 시스템) 내부에서 라우팅에 사용됩니다.
- RIP보다 빠르고 큰 네트워크에 적합합니다.

RIP vs OSPF 비교 (자주 출제):
- **RIP**: 거리 벡터, 홉 카운트, 최대 15홉, 소규모
- **OSPF**: 링크 상태, 다익스트라, 대규모

키워드 "**링크 상태, Dijkstra**"는 **OSPF**입니다.

## 8번

아래 내용의 각각의 설명에 대한 답을 작성하시오.

(1) 조인에 참여하는 두 릴레이션의 속성 값을 비교하여 조건을 만족하는 튜플만 반환한다.
(2) 조건이 정확하게 '=' 등호로 일치하는 결과를 반환한다.
(3) ((2)) 조인에서 조인에 참여한 속성이 두 번 나오지 않도록 중복된 속성을 제거한 결과를 반환한다.

**정답:** (1) 세타 조인 / (2) 동등 조인 / (3) 자연 조인

**해설:**
**조인(Join)**의 종류를 묻는 문제입니다. 조인은 두 테이블을 연결하는 연산입니다.

- **세타 조인(Theta Join)**: `=, >, <, >=` 등 **다양한 비교 연산자**로 조건을 거는 가장 일반적인 조인입니다. (가장 넓은 개념)
- **동등 조인(Equi Join)**: 세타 조인 중에서 **'=' 등호만** 사용하는 조인입니다. 조인 조건이 같은 값일 때만 연결합니다.
- **자연 조인(Natural Join)**: 동등 조인에서 **중복되는 속성(공통 열)을 하나로 합쳐 제거**한 결과입니다. 같은 열이 두 번 안 나오게 정리한 것입니다.

관계: **세타 조인 ⊃ 동등 조인 ⊃ 자연 조인** (점점 더 구체적/제한적).

암기 팁: "**세타=모든 비교, 동등==만, 자연=중복속성 제거**".

## 9번

다음은 운영체제 페이지 순서를 참고하여 할당된 프레임의 수가 3개일 때 LRU와 LFU 알고리즘의 페이지 부재 횟수를 작성하시오.

페이지 참조 순서: 1, 2, 3, 1, 2, 4, 1, 2, 5, 7

**정답:** (1) LRU: 6 / (2) LFU: 6

**해설:**
**페이지 교체 알고리즘** 문제입니다. 프레임(메모리 칸)이 3개뿐일 때, 새 페이지를 넣으려면 기존 페이지 하나를 내보내야 합니다. **페이지 부재(Page Fault)**는 찾는 페이지가 메모리에 없어서 새로 불러오는 횟수입니다.

- **LRU(Least Recently Used)**: **가장 오래 안 쓴(최근에 사용 안 한)** 페이지를 내보냅니다.
- **LFU(Least Frequently Used)**: **가장 적게 사용된(사용 횟수가 적은)** 페이지를 내보냅니다.

LRU 추적 (프레임 3개, *는 부재):
```
1*: [1]
2*: [1,2]
3*: [1,2,3]
1 : [1,2,3]  (있음, hit) → 1 최근 사용
2 : [1,2,3]  (있음, hit) → 2 최근 사용
4*: 가장 오래 안 쓴 3 제거 → [1,2,4]
1 : hit
2 : hit
5*: 가장 오래 안 쓴 4 제거 → [1,2,5]
7*: 가장 오래 안 쓴 1 제거 → [7,2,5]
```
부재: 1,2,3,4,5,7 → **6회**

LFU도 같은 방식으로 사용 빈도가 낮은 것을 내보내며 계산하면 부재 **6회**가 나옵니다(1,2는 자주 쓰여 남고, 빈도 낮은 것들이 교체됨).

핵심: LRU는 "**언제 썼나(시간)**", LFU는 "**몇 번 썼나(횟수)**"가 교체 기준입니다. 표를 그려 한 단계씩 추적하는 것이 정확합니다.

## 10번

아래 JAVA언어 코드의 실행 순서를 중복 번호 없이 작성하시오.

```java
class Parent {
    int x, y;
    Parent(int x, int y) {   // ①
        this.x = x;
        this.y = y;
    }
    int getT() {             // ②
        return x*y;
    }
}

class Child extends Parent {
    int x;
    Child(int x) {           // ③
        super(x+1, x);
        this.x = x;
    }
    int getT(int n){         // ④
        return super.getT()+n;
    }
}

class Main {
    public static void main(String[] args) {   // ⑤
        Parent parent = new Child(3);          // ⑥
        System.out.println(parent.getT());     // ⑦
    }
}
```

실행 순서: 5 → ( ) → ( ) → ( ) → ( ) → ( )

**정답:** 6 3 1 7 2

**해설:**
**프로그램 실행 순서**를 추적하는 문제입니다. **상속에서는 부모 생성자가 먼저 실행**된다는 점이 핵심입니다.

흐름:
1. **⑤** main 시작
2. **⑥** `new Child(3)` 객체 생성 시작
3. **③** Child(int x) 생성자 진입 → 첫 줄 `super(x+1, x)`로 **부모 생성자 호출**
4. **①** Parent(int x, int y) 생성자 실행 (x, y 설정). 부모 생성자가 끝나야 자식 생성자 나머지가 실행됨
   - (Parent 생성 후 Child 생성자의 `this.x = x`가 실행되지만 별도 번호 없음)
5. **⑦** `parent.getT()` 호출
6. **②** getT() 실행 → x*y 반환
   - (주의: Child의 `getT(int n)`은 매개변수가 있어 **오버로딩**된 별개 메서드입니다. 인자 없는 `getT()`를 불렀으므로 부모의 ②가 실행됨)

순서: 5 → 6 → 3 → 1 → 7 → 2

핵심: "**객체 생성 시 자식 생성자 → super()로 부모 생성자 먼저 실행 → 부모 끝나고 자식 나머지**" 순서. 그리고 `getT()`(인자 없음)와 `getT(int n)`(인자 있음)은 다른 메서드라는 오버로딩 함정에 주의하세요.

- 숫자: `(글자-'0'+3) % 10 + '0'` → +3 이동

**정답:** Abstract Factory

**해설:**
**추상 팩토리(Abstract Factory)**는 GoF **생성 패턴** 중 하나입니다.

- **서로 관련 있는 객체들의 "묶음(군)"을 통째로 생성**하는 인터페이스를 제공합니다.
- 구체적인 클래스 이름에 의존하지 않고, 상황에 맞는 객체 군을 만들어냅니다.
- 비유: "겨울 테마 가구 세트(의자+책상+조명)"를 한꺼번에 만드는 공장과, "여름 테마 세트"를 만드는 공장을 따로 두는 것. 어떤 공장을 쓰느냐에 따라 일관된 세트가 생성됩니다.
- 별칭으로 "**Kit**"라고도 불립니다.

비교: **팩토리 메서드(Factory Method)**는 객체 하나의 생성을 서브클래스에 위임하는 패턴이고, **추상 팩토리**는 **관련된 여러 객체 군**을 만드는 더 큰 개념입니다.

키워드 "**연관된 객체 군, kit**"는 **Abstract Factory**입니다.
