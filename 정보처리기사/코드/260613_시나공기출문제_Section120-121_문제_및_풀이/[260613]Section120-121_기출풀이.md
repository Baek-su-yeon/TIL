### C 문제 - 예제 1.c

```c
#include <stdio.h>
main() {
	int a = 50;
	int *b = &a;
	*b = *b + 20;
	printf("%d, %d\n", a, *b);
	char *s;
	s = "gilbut";
	for (int i = 0; i < 6; i += 2) {
		printf("%c, ", s[i]);
		printf("%c, ", *(s + i));
		printf("%s\n", s + i);
	}
}
```

```c
// 예상 정답

// 진짜 정답
70, 70
g, g, gilbut
l, l, lbut
u, u, ut
```

[오답풀이-gpt]

- **b와 s는 a와 ‘g(문자열 첫번째)’의 주소, *b와 *s는 그 주소에 있는 int 값과 문자 값을 의미**
- b는 int형 데이터를 가리키는 포인터 변수이고, 그 안에는 a의 주소가 저장된다.
즉 b가 a의 주소를 가지고 있고 *b는 그 주소의 값
    
    a    → 50
    &a   → a의 주소, 예: 100번지
    b    → a의 주소, 예: 100번지
    *b   → b가 가리키는 곳의 값 → a의 값 → 50
    *b = *b + 20 → a = a + 20 → a = 70, *b = 70
    
- 참고로 printf("%s\n", s + i); 는 i번째부터 s의 모든 글자를 출력하겠다
    
    
    | 인덱스 | 0 | 1 | 2 | 3 | 4 | 5 | 6 |
    | --- | --- | --- | --- | --- | --- | --- | --- |
    | 문자열 | g | i | l | b | u | t | \0 |
    | 주소 | s+0 | s+1 | s+2 | s+3 | s+4 | s+5 | s+6 |
    
    | s[0] | g | s[2] | l | s[4] | u |
    | --- | --- | --- | --- | --- | --- |
    | *(s+0) | g | *(s+2) | l | *(s+4) | u |
    | s+0 | gilbut | s+2 | lbut | s+4 | ut |

### 기출 따라잡기 - 문제 1.c

```c
#include <stdio.h>
main() {
	char *p = "KOREA";
	printf("%s\n", p);
	printf("%s\n", p + 3);
	printf("%c\n", *p);
	printf("%c\n", *(p + 3));
	printf("%c\n", *p + 2);
}
```

```c
// 예상 정답
KOREA
EA
K
E
M
```

- p는 KOREA 문자열의 가장 첫글자인 K의 주소이므로 printf("%s\n", p);는 전체 문자열 출력
- p + 3는 3번째 인덱스인 E의 주소이므로 printf("%s\n", p + 3);는 E부터 끝까지 출력
- *p는 문자열의 가장 첫글자인 K 그 자체를 뜻하므로 printf("%c\n", *p);는 K만 출력
- *(p + 3)은  3번째 인덱스인 E 그 차체를 뜻하므로 printf("%c\n", *(p + 3)); E만 출력
- *p의 값인 K 자체에 +2하므로 M이 출력 (아스키코드로 변환되어 계산 후 %c에 맞게 출력)

### 기출 따라잡기 - 문제 2.c

```c
#include <stdio.h>
int main() {
	int ary[3];
	int s = 0;
	*(ary + 0) = 1;
	ary[1] = *(ary + 0) + 2;
	ary[2] = *ary + 3;
	for (int i = 0; i < 3; i++)
		s = s + ary[i];
	printf("%d", s);
}
```

```c
// 예상 정답
8
```

- *(ary + 0)는 ary[0]의 값을 뜻하므로 ary[0]에 1이 할당
- *(ary + 0) + 2는 ary[0]의 값 + 2이므로 ary[1] = 3
- *ary + 3은 ary의 가장 첫번째 배열의 값 + 3이므로  ary[2] = 1 + 3 = 4
- s는 ary 배열의 총 합이므로 8이 출력
- (ary + 0) = 1은 ary[0]에 1을 저장하는 코드이고, 이후 *(ary + 0), *ary는 ary[0]의 값을 읽어 계산에 사용하는 표현이다

### 기출 따라잡기 - 문제 3.c

```c
#include <stdio.h>
int main() {
	int* array[3];
	int a = 12, b = 24, c = 36;
	array[0] = &a;
	array[1] = &b;
	array[2] = &c;

}
```

```c
// 예상 정답

// 진짜 정답
37
```

- array의 각 배열은 a,b,c의 주소를 가지고 있음
- *array[1]은 b의 값을 뜻하므로 24

[오답풀이 - gpt]

- *array는 array[0]의 값이 뭐냐, **array는 array[0]의 주소에 있는 값이 뭐냐라는 말
    
    array[1]  → &b
    *array[1] → b의 값 → 24
    
    array     → &array[0]
    *array    → array[0] → &a
    **array   → *(&a) → a의 값 → 12

### 기출 따라잡기 - 문제 4.c

```c
#include <stdio.h>
int main( ) {
	int a[4] = { 0, 2, 4, 8 };
	int b[3];
	int* p;
	int sum = 0;
	for (int i = 1; i < 4; i++) {
		p = a + i;
		b[i - 1] = *p - a[i - 1];
		sum = sum + b[i - 1] + a[i];
	}
	printf("%d", sum);
}
```

```c
// 예상 정답
22
```

- p는 a[1], a[2], a[3]을 순차적으로 가르키므로 *p도 각 배열의 값을 가르키게됨
- 따라서 b = {2, 2, 4}가 될 것임
- sum = 0 + 2 + 2 → 4 + 2 + 4 → 10 + 4 + 8 → 22

### 02 C문제 - 예제 1.c

```c
#include <stdio.h>
struct jsu {
	char nae[12];
	int os, db, hab, hhab;
};
int main() {
	struct jsu st[3] = { {"데이터1", 95, 88}, {"데이터2", 84, 91}, {"데이터3", 86, 75} };
	struct jsu* p;
	p = &st[0];
	(p + 1)->hab = (p + 1)->os + (p + 2)->db;
	(p + 1)->hhab = (p + 1)->hab + p->os + p->db;
	printf("%d", (p + 1)->hab + (p + 1)->hhab);
}
```

```c
// 예상 정답

// 진짜 정답
501
```

- jsu라는 구조체는 char 12칸짜리 nae 배열과 4개의 int 변수를 가짐
- st[3]은 jsu 구조체 3개로 이뤄진 배열로 각 구조체에 nae와 os, db까지 할당됨
- p는 jsu 구조체 형식을 가리키는 포인터 변수이며 st[0]의 주소가 할당됨

[오답풀이-gpt]

- `->` 는 구조체 포인터가 가리키는 구조체의 멤버에 접근할 때 쓰는 연산자
- `p -> os`와 `(*p).os`는 같은 말이며 현재 이 코드에서 `p -> os` 는 `st[0].os` 를 뜻함
- `(p + 1)->hab = (p + 1)->os + (p + 2)->db` 는 `st[1].hab = st[1].os + st[2].db = 84 + 75 = 159`
- `(p + 1)->hhab = (p + 1)->hab + p->os + p->db` 는 `st[1].hhab = st[1].hab + st[0].os + st[0].db = 159 + 95 + 88 = 342`

### 기출 따라잡기 - 문제 3.c

```c
#include <stdio.h>

struct dat {
	int x;
	int y;
};

int main( ) {
	struct dat a[] = {{1, 2}, {3, 4}, {5, 6}};
	struct dat* ptr = a;
	struct dat** pptr = &ptr;
	(*pptr)[1] = (*pptr)[2];
	printf("%d 그리고 %d", a[1].x, a[1].y);
	return 0;
}
```

```c
// 예상 정답
5 그리고 6
```

- (*pptr)[1] = (*pptr)[2]로 인해 a = {{1, 2},{5, 6},{5, 6}}이 됨
- a[1].x = 5, a[1].y = 6

[오답풀이-gpt]

- 잘못 이해한 것 struct dat** pptr은 struct dat* 변수의 포인터 변수를 선언하는 것임!!!!! 뭔 **이거라고 두번 타고 들어가라는게 아니라 ptr이 struct dat* 변수이므로 이 변수의 포인터라는 것
- 그래서 결국 아래와 같음
    
    pptr
    ↓
    ptr
    ↓
    a[0]        a[1]        a[2]
    {1, 2}      {3, 4}      {5, 6}
    
    ptr  = a[0]의 주소를 담는 상자
    pptr = ptr 상자의 주소를 담는 상자
