### 02 C문제 - 예제 1.c

```c
#include <stdio.h>
int factorial(int n);
main() {
	int (*pf)(int);
	pf = factorial;
	printf("%d", pf(3));
}
int factorial(int n) {
	if (n <= 1)
		return 1;
	else
		return n * factorial(n - 1);
}
```

```c
// 예상 정답
6
```

<aside>

**내 풀이**

재귀함수로 결국 1부터 3까지 factorial 계산하는 것 (근데 감으로 푼거라 이게 왜 재귀함수처럼 동작하는 건지 모름)

</aside>

<aside>

**GPT 풀이**

| `int (*pf) (int);` | `int *pf (int);` |
| --- | --- |
| 함수 포인터 | int 포인터를 반환하는 함수 |

`int (*pf) (int);` 의 의미
pf는 i**nt형 매개변수 하나**를 받고, **int형 값을 반환**하는 함수를 **가리킬 수 있는** **포인터 변수**

만약 괄호가 없다면 `int *pf (int);` 의 의미
pf는 **int를 매개변수**로 받고, **int 포인터를 반환**하는 pf 라는 함수명을 가진 **함수**

`pf = factorial;` 이 문장을 통해 pf에 factorial 함수의 주소를 저장

</aside>

### 기출 따라잡기 - 문제 1.c

```c
#include <stdio.h>
void align(int a[]) {
	int temp;
	for (int i = 0; i < 4; i++)
		for (int j = 0; j < 4 - i; j++)
			if (a[j] > a[j+1]) {
				temp = a[j];
				a[j] = a[j+1];
				a[j+1] = temp;
			}
}
main() {
	int a[] = { 85, 75, 50, 100, 95 };
	align(a);
	for (int i = 0; i < 5; i++)
		printf("%d ", a[i]);
}
```

```c
// 예상 정답
50 75 85 95 100 
```

<aside>

**내 풀이**

align 함수는 int형의 배열을 입력 받아 오름차순 정렬하는 함수(버블정렬..?)이므로 출력 시 오름차순으로 정렬된 a 배열이 순차적으로 출력됨

</aside>

### 기출 따라잡기 - 문제 2.c

```c
#include <stdio.h>
#include <string.h>
void inverse(char *str, int len) {
	for(int i = 0, j = len - 1; i < j; i++, j--) {
		char ch = str[i];
		str[i] = str[j];
		str[j] = ch;
	}
}
int main() {
	char str[100] = "ABCDEFGH";
	int len = strlen(str);
	inverse(str, len);
	for(int i = 1; i < len; i += 2) {
		printf("%c", str[i]);
	}
	return 0;
}
```

```c
// 예상 정답
BDFH

// 진짜 정답
// 뒤집는 함수라고 적어두고 왜 안뒤집었는지,,
GECA
```

<aside>

**내 풀이**

inverse 함수는 함수의 str이 가리키는 문자열을 거꾸로 **뒤집는 함수**이다.

하지만 main의 for문에 의해 출력 시 홀수 인덱스의 문자만 출력된다.

### 기출 따라잡기 - 문제 5.c

```c
#include <stdio.h>
int func( ) {
	static int x = 0;
	x += 2;
	return x;
}
int main( ) {
	int x = 0;
	int sum= 0;
	for(int i = 0; i < 4; i++) {
		x++;
		sum += func( );
	}
	printf("%d", sum);
	return 0;
}
```

```c
// 진짜 정답
20
```

<aside>

**GPT 풀이**

**static 지역 변수**
- 함수 안에서 선언되지만 **함수가 끝나도 값이 유지**
- 초기화는 **한 번**만, 다음 호출 때 이전 값을 이어서 사용

**main 함수의 x와 func 함수의 x는 전혀 다른 변수**

| 반복 | main의 `x++` 후 | `func()` 안 static x | `func()` 반환값 | sum |
| --- | --- | --- | --- | --- |
| 1회차 | 1 | 2 | 2 | 2 |
| 2회차 | 2 | 4 | 4 | 6 |
| 3회차 | 3 | 6 | 6 | 12 |
| 4회차 | 4 | 8 | 8 | 20 |
</aside>

### 기출 따라잡기 - 문제 6.c

```c
#include <stdio.h>
int isPrime(int number) {
	for (int i = 2; i < number; i++)
		if (number % i == 0) return 0;
	return 1;
}
int main() {
	int number = 13195;
	int max_div = 0;
	for (int i = 2; i < number; i++)
		if (isPrime(i) == 1 && number % i == 0) max_div = i;
	printf("%d", max_div);
}
```

```c
// 진짜 정답
29
```

<aside>

**내 풀이**

isPrime은 약수가 1과 자신 뿐인 소수를 찾는 함수이다.

main 함수의 for문 안에 if 절은 i의 약수가 1과 i 뿐인 소수를 찾으면서 해당 숫자가 13195의 약수인지 확인하는 구문으로 즉 13195의 약수 중 소수의 개수를 찾는 코드이다.

</aside>

<aside>

**GPT 풀이**

일단, 13195의 약수 중 소수의 개수가 아닌 조건을 만족하는 i를 계속 저장하는 것
즉 13195의 가장 큰 소인수를 출력함

그러므로 실제 시험에서는 소인수 분해해서 가장 큰 숫자 찾을 것
13195 = 5 * 7 * 13 * 29 이므로 29가 출력됨

</aside>

### 기출 따라잡기 - 문제 7.c

```c
#include <stdio.h>
main() {
	int res = mp(2, 10);
	printf("%d", res);
}
int mp(int base, int exp) {
	int res = 1;
	for (int i = 0; i < exp; i++)
		res *= base;
	return res;
}
```

```c
// 예상 정답
1024
```

<aside>

**내 풀이**

mp는 지수 계산함수로 base를 exp만큼 곱하는 함수이다.
즉 2의 10승 계산 결과인 1024 출력

### 기출 따라잡기 - 문제 8.c

```c
#include <stdio.h>
void func(int** arr, int size) {
	for(int i = 0; i < size; i++) {
		*(*arr + i) = (*(*arr + i) + i) % size;
	}
}
int main( ){
	int arr[] = {3, 1, 4, 1, 5};
	int* p = arr;
	int** pp = &p;
	int num = 6;
	func(pp, 5);
	num = arr[2];
	printf("%d", num);
	return 0;
}
```

```c
// 진짜 정답
1
```

<aside>

**GPT 풀이**

pp → p → arr 이 순서로 현재 가리키고 있음

함수 안의 arr → p를 가리킴, 왜냐면 pp가 입력 됐으니까
*arr는 p의 값을 가리킴, p에는 arr[0]의 주소가 있으니까 즉 *arr → main 함수 arr[0]의 주소
*arr+i 는 주소를 0부터 끝까지 순회하겠다는 말
*(*arr+i)는 **해당 주소의 값**을 보겠다는 말

즉! *(*arr + i) = (*(*arr + i) + i) % size → arr[i] = (arr[i] + i) % size
결국 main 함수의 arr = {3, 2, 1, 4, 4}
num = arr[2] = 1

</aside>
