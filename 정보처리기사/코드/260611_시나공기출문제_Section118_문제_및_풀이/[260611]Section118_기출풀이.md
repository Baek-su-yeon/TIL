### [SECTION 118] 02 C문제 - 예제 1.c

```c
#include <stdio.h>
main()
{
	int i, j, k;
	scanf("%d %d", &i, &j);
	k = i + j;
	printf("%d\n", k);
}
```

```c
// 예상 정답
// i, j에 2와 3이 입력됐다고 가정
5
// 줄바꿈으로 인해 커서 밑으로 내려와 있을 거임
```

- scanf()는 입력을 받는 함수이며, 현재 2개의 정수 타입 입력을 받아서 각각 i와 j에 할당한다. k에 i와 j의 합이 할당되고, printf() 함수에서 지정된 출력형식(정수타입의 값과 줄바꿈)으로 k의 값이 출력된다.
- &는 참조 연산자로 i와 j의 제일 처음 주소를 가르킨다

### [SECTION 118] 기출 따라잡기_문제 2.c

```c
#include <stdio.h>
main()
{
	int i = 10, j = 10, k = 30;
	i /= j;
	j -= i;
	k %= j;
	printf("%d, %d, %d\n", i, j, k);
}
```

```c
//예상 정답
1, 9, 3
// 줄바꿈으로 커서는 아래에 존재
```

- `i /= j`  → `i = i / j` 이므로 `i = 10 / 10 = 1`
- `j -= i` → `j = j - i` 이므로 `j = 10 - 1 = 9`
- `k %= j` → `k = k % j` 이고, 나머지 연산이므로 `k = 30 % 9 = 3`

### [SECTION 118] 기출 따라잡기_문제 4.c

```c
#include <stdio.h>
main( )
{
	int result, a = 100, b = 200, c = 300;
	result = a < b ? b++ : --c;
	printf("%d, %d, %d\n", result, b, c);
}
```

```c
// 예상 정답, 줄바꿈은 이제 알아서,,있을것이라고,, 생각해,,
200, 201, 299
```

- result는 a < b 라는 조건문이 참일 경우 b 값이 할당되고, 거짓일 경우 —c 값이 할당된다.
- b++은 해당 줄의 모든 연산이 끝난 후에 b에 1을 더하는 연산자로 마지막에 b는 201이 된다.
- —c는 해당 줄의 모든 연산이 시작 되기 전 c에 1을 빼는 연산자로, 조건이 거짓은 아니지만 이미 -1이 됐으므로 299가 된다.

### [SECTION 118] 기출 따라잡기_문제 5.c

```c
#include <stdio.h>
main() {
	int i, j;
	scanf("%o#%x", &i, &j);
	printf("%d %d", i, j);
}
```

```c
// 예상 정답
// 오 이거 헤깔림
```

[오답풀이]

- %o는 8진수로 문자열 ‘0’ ~ ‘7’를 입력 받음
- %x는 16진수로 ('0'~'9', 'a'~'f', 'A'~'F')을 입력 받음
- 출력은 %d로 인해 10진수로 출력될 것임
- 8진수 → 10진수: 각 자릿수에 8의 거듭제곱 곱한 후 모두 더하기
- 16진수 → 10진수: 각 자릿수에 16의 거듭제곱 곱한 후 모두 더하기

### [SECTION 118] 기출 따라잡기_문제 6.c

```c
#include <stdio.h>
main()
{
	int j = 024, k = 24, L = 0x24, hap;
	hap = j + k + L;
	printf("%d, %d, %d, %d\n", j, k, L, hap);
}
```

```c
//예상 정답
//이것도 아예 못맞춤
```

[오답풀이]

- `j = 024`는 8진수 24를 뜻함, 즉 숫자 앞에 0을 붙일 경우 C언어에서는 8진수로 해석
- `L = 0x24`는 16진수 24를 뜻함, 즉 숫자 앞에 0x를 붙일 경우 C언어에서는 16진수로 해석
- j를 10진수로 바꾸면 `j = (2 * 8) + (4 * 1) = 20`
- L을 10진수로 바꾸면 `L = (2 * 16) + (4 * 1) = 36`
- `hap = 20 + 24 + 36 = 80`
- 최종 출력은 `20, 24, 36, 80`

### [SECTION 118] 기출 따라잡기_문제 7.c

```c
#include <stdio.h>
int main( ) {
	int x = 7, y = 4, z;
	z = y % 3 < 3 ? 2 : 1;
	z = z & z >> 1;
	z = x > 5 && z <= 3 ? z * x : z / x;
	printf("%d", z);
	return 0;
}
```

```c
//예상정답
7
```

- y % 3 = 4 % 3 = 1 이므로 1 < 3 조건이 만족하여 첫번째 z = 2
- z & z >> 1에서 &는 비트 and 연산자로 10진수 2진수로 변환하면 10이다. 10 & 10은 각 자리의 비트끼리만 and 연산을 하면되므로 10 & 10 = 10 이다.
- >> 1은 비트를 오른쪽으로 한칸 옮기라는 의미이므로 두번째 z = 1이된다.
- x > 5 && z <= 3? 에서  x > 5는 7 > 5로 참, z ≤ 3은 1 ≤ 3으로 참, 결국 두 조건이 참이므로 세번째 z = z * x = 1 * 7 = 7이다.

### [SECTION 118] 기출 따라잡기_문제 10.c

```c
#include <stdio.h>
main()
{
	int a = 5, b = 10, c = 15, d = 30, result;
	result = a * 3 + b > d || c - b / a <= d && 1;
	printf("%d\n", result);
}
```

```c
// 예상 정답
1
```

- a * 3 + b > d에서 a * 3 + b = 5 * 3 + 10 = 25 이므로 25 > 30은 거짓
- c - b / a <= d에서 c - b / a = 15 - 10 / 5 = 13 이므로 13 ≤ 30은 참
- a * 3 + b > d || c - b / a <= d 는 거짓 or 참 이므로 참 = 1이며 result = 1 && 1 = 1 이다.

```java
import java.util.Scanner;
```


### [SECTION 118] 기출 따라잡기_문제 8.java

```java
public class Test {
	public static void main(String[] args) {
		int w = 3, x = 4, y = 3, z = 5;
		if((w == 2 | w == y) & !(y > z) & (1 == x ^ y != z)) {
			w = x + y;
			if(7 == x ^ y != w)
				System.out.println(w);
			else
				System.out.println(x);
		}
		else {
			w = y + z;
			if(7 == y ^ z != w)
				System.out.println(w);
			else
				System.out.println(z);
		}
	}
}
```

```java
// 예상정답

```

[오답풀이]

- **연산자 우선 순위**
    
    
    | 순위 | 연산자 | 역할 |
    | --- | --- | --- |
    | 1 | `!` | 논리 부정 |
    | 2 | `>, <` | 크기 비교 |
    | 3 | `==, !=`  | 동등 비교 |
    | 4 | `&` | AND |
    | 5 | `^` | XOR (**두 값이 서로 다르면 true**, **같으면 false)** |
    | 6 | `|` | OR |
- 자바는 뭔가 코드가 보기 싫게 생겻다.
- (w == 2 | w == y)은 거짓 or 참이므로 참
- !(y > z)은 거짓이나 “!”으로 인해 결국 참
- (1 == x ^ y != z)는 1 == x가 거짓, y != z가 참이므로 거짓 ^ 참을 XOR 연산하면 참
- 즉,  (w == 2 | w == y) & !(y > z) & (1 == x ^ y != z)는 참 & 참 & 참이므로 if문 안으로 진입
- if(7 == x ^ y != w)는 if(7 == 4 ^ 3 != 7) → if(거짓 ^ 참) → if(참)이므로 최종적으로 w 값인 7이 출력된다.
