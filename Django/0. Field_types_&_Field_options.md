## 1. Model Field Types

| 필드 타입 | 그룹 | 설명 |
| :--- | :--- | :--- |
| **`AutoField`** | 정수/ID | 자동 증가(auto-incrementing)하는 정수 기본 키(Primary Key)입니다. |
| **`BigAutoField`** | 정수/ID | 64비트 정수형 자동 증가 기본 키입니다. (일반 `AutoField`보다 큰 범위) |
| **`BigIntegerField`** | 정수 | -9223372036854775808부터 9223372036854775807까지의 값을 갖는 64비트 정수 필드입니다. |
| **`BinaryField`** | 바이너리 | 원시(raw) 바이너리 데이터를 저장합니다. |
| **`BooleanField`** | 논리 | True/False 값을 저장하는 필드입니다. (기본 폼 위젯: Checkbox) |
| **`CompositePrimaryKey`** | ID | 튜플로 정의된 다중 필드를 지원하는 복합 기본 키 필드입니다. |
| **`CharField`** | 문자열 | 문자열(string)을 저장합니다. **`max_length`가 필수**입니다. |
| **`DateField`** | 날짜/시간 | Python의 `datetime.date` 객체를 저장합니다. (날짜만) |
| **`DateTimeField`** | 날짜/시간 | Python의 `datetime.datetime` 객체를 저장합니다. (날짜와 시간) |
| **`DecimalField`** | 숫자 | 고정 소수점 숫자를 저장합니다. **`max_digits`와 `decimal_places`가 필수**입니다. |
| **`DurationField`** | 날짜/시간 | 시간의 길이(`datetime.timedelta`)를 저장합니다. |
| **`EmailField`** | 문자열 | 이메일 주소를 저장합니다. 내부적으로 `CharField`를 사용하며 이메일 형식 유효성 검사가 추가됩니다. |
| **`FileField`** | 파일 | 파일 업로드 정보를 저장하는 필드입니다. (데이터베이스에는 파일 경로만 저장) |
| **`FilePathField`** | 문자열 | 특정 디렉터리 경로 내의 파일 이름 문자열을 저장합니다. |
| **`FloatField`** | 숫자 | 부동 소수점(Floating-point) 숫자를 저장합니다. |
| **`GeneratedField`** | 숫자 | 데이터베이스에서 계산되어 저장되는 필드입니다. |
| **`GenericIPAddressField`** | 문자열 | IPv4 또는 IPv6 주소를 저장합니다. |
| **`ImageField`** | 파일 | 이미지 파일을 저장합니다. `FileField`를 상속받으며 이미지 유효성 검사 기능이 추가됩니다. |
| **`IntegerField`** | 정수 | -2147483648부터 2147483647까지의 값을 갖는 32비트 정수 필드입니다. |
| **`JSONField`** | 구조화 | 구조화된 데이터(JSON 형식)를 저장합니다. |
| **`PositiveBigIntegerField`** | 정수 | 0 또는 양의 64비트 정수 필드입니다. |
| **`PositiveIntegerField`** | 정수 | 0 또는 양의 정수 필드입니다. |
| **`PositiveSmallIntegerField`** | 정수 | 0 또는 양의 작은 정수(2바이트) 필드입니다. |
| **`SlugField`** | 문자열 | 슬러그(slug)를 저장합니다. 주로 URL에 사용할 수 있는 짧은 레이블을 저장하며, **`max_length`가 선택적**입니다. |
| **`SmallAutoField`** | 정수/ID | 0부터 32767까지의 값을 갖는 작은 정수 자동 증가 기본 키입니다. |
| **`SmallIntegerField`** | 정수 | -32768부터 32767까지의 값을 갖는 작은 정수(2바이트) 필드입니다. |
| **`TextField`** | 문자열 | 길이가 긴 텍스트를 저장합니다. (DB가 지원하는 최대 길이) |
| **`TimeField`** | 날짜/시간 | Python의 `datetime.time` 객체를 저장합니다. (시간만) |
| **`URLField`** | 문자열 | URL을 저장합니다. 내부적으로 `CharField`를 사용하며 URL 형식 유효성 검사가 추가됩니다. |
| **`UUIDField`** | 문자열/ID | 범용 고유 식별자(UUID)를 저장합니다. |
| **`ForeignKey`** | 관계 | 다 대 일(Many-to-one) 관계를 정의합니다. **참조할 모델이 필수**입니다. |
| **`ManyToManyField`** | 관계 | 다 대 다(Many-to-many) 관계를 정의합니다. **참조할 모델이 필수**입니다. |
| **`OneToOneField`** | 관계 | 일 대 일(One-to-one) 관계를 정의합니다. **참조할 모델이 필수**입니다. |

## 2. Model Field Options (모든 필드 타입에 공통 적용되는 옵션)

| 옵션명 | 설명 | 비고 |
| :--- | :--- | :--- |
| **`null`** | `True`인 경우, 데이터베이스에 빈 값을 **NULL**로 저장할 수 있도록 허용합니다. 기본값은 `False`입니다. | `CharField`, `TextField` 같은 문자열 기반 필드는 일반적으로 `NULL` 대신 빈 문자열(`''`)을 사용하는 것이 권장됩니다. |
| **`blank`** | `True`인 경우, **폼(Form) 유효성 검사**에서 해당 필드를 빈 값으로 제출하는 것을 허용합니다. 기본값은 `False`입니다. | **데이터베이스**와 관련된 `null`과 달리, `blank`는 **폼 검증**과 관련됩니다. |
| **`choices`** | 필드에 사용될 선택 항목(Choice)을 정의합니다. 선택 항목이 주어지면 모델 유효성 검사 시 이를 강제하며, 기본 폼 위젯은 텍스트 필드 대신 **선택 상자(select box)**가 됩니다. | `(실제 값, 사람이 읽기 쉬운 이름)` 형태의 매핑 또는 이터러블로 전달합니다. |
| **`db_column`** | 이 필드에 사용할 데이터베이스 열(Column)의 이름을 지정합니다. 지정하지 않으면 필드 이름이 사용됩니다. | |
| **`db_comment`** | 이 필드에 사용할 데이터베이스 열에 대한 주석(Comment)을 지정합니다. 데이터베이스에 직접 접근하는 사람들을 위해 필드를 문서화하는 데 유용합니다. | |
| **`db_default`** | 이 필드에 대한 데이터베이스에서 계산된 **기본값**을 지정합니다. 리터럴 값이나 `Now()`와 같은 데이터베이스 함수가 될 수 있습니다. | Python 코드에서 인스턴스를 생성할 때는 `default`가 `db_default`보다 우선합니다. |
| **`db_index`** | `True`인 경우, 이 필드에 대한 데이터베이스 인덱스가 생성됩니다. | 가능하면 `Meta.indexes` 옵션을 대신 사용하는 것이 좋습니다. |
| **`default`** | 필드의 기본값을 설정합니다. 새 모델 인스턴스가 생성될 때 값이 제공되지 않으면 이 값이 사용됩니다. | `ForeignKey`와 같은 필드의 기본값은 참조하는 필드의 값이어야 합니다. |
| **`editable`** | `False`인 경우, 이 필드는 Admin이나 `ModelForm`에 표시되지 않으며 모델 유효성 검사 시 건너뜁니다. 기본값은 `True`입니다. | |
| **`error_messages`** | 필드가 발생시키는 기본 오류 메시지를 재정의할 수 있습니다. 덮어쓸 오류 메시지에 해당하는 키를 가진 딕셔너리를 전달합니다. | |
| **`help_text`** | 폼 위젯과 함께 표시될 추가적인 "도움말" 텍스트입니다. 필드가 폼에 사용되지 않더라도 문서화에 유용합니다. | 자동으로 생성된 폼에서는 HTML 이스케이프가 되지 않습니다. |
| **`primary_key`** | `True`인 경우, 이 필드는 모델의 기본 키(Primary Key)가 됩니다. | 모델에 `primary_key=True`를 지정하지 않으면, Django가 자동으로 기본 키 필드를 추가합니다. |
