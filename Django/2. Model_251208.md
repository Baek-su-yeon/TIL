## Model이란?
- 데이터베이스와 python 클래스(객체)로 추상화된 형태로 상호작용
    ### Model을 통한 DB 관리
    |파일명|기능|
    |:---|:---|
    |urls.py|사용자 요청의 시작점|
    |views.py|요청을 처리하고 modles.py를 통해 데이터 관리|
    |models.py|**데이터베이스 정의, 데이터베이스와 상호작용**|
    |templates|views.py로부터 받은 데이터로 화면 구성|
    
    ### Model class
    ```python
    # python
    from django.db import models

    class Article(models.Model): # 테이블명
        # 필드(열)이름 = models.Field_types(Field options)
        title = models.CharField(max_length=10)
        content = models.TextField()
    ```

## Migrations
- model 클래스의 변경사항(필드 생성, 수정 삭제 등)을 DB에 최종 반영하는 방법
- 즉, 우리는 파이썬으로 DB를 설계하는데 실제 DB는 SQL만 이해할 수 있으므로 중간에 번역해주는 것
    ### 과정
    1. model.py에서 model 설계
    2. migration 파일(예: 0001_initial.py) 생성
    ```bash
    python manage.py makemigrations
    ```
    3. DB 반영
    ```python
    python manage.py migrate
    ```

## Admin site
- Django가 추가 설치 및 설정 없이 자동으로 제공하는 관리자 인터페이스 (추후 바꿔야함)
    ### 관리자 계정 생성
    📌 비밀번호 입력 시 원래 보이지 않음
    ```bash
    python manage.py createsuperuser
    ```
    ### 모델 추가
    ```python
    # python
    # admin.py
    from django.contrib import admin
    from .models import Article # 내가 만든 모델

    admin.site.register(Article) # admin 페이지에 Article 목록 추가
    # 내가 설계한대로 admin 페이지에서 내용 추가 가능
    ```
