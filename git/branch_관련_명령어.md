# 🛠️ Git Workflow 및 주요 명령어

본 프로젝트는 효율적인 협업과 환경 전이를 위해 Git 브랜치 전략을 적극 활용하며, 최신 Git 커맨드(`switch`)를 표준으로 사용합니다.

## 1. 작업 저장 및 원격 동기화 (Office → Remote)
사무실 PC에서 작업한 안드로이드 AI 엔진 코드를 원격 저장소에 안전하게 백업할 때 사용합니다.

* **`git status`**: 현재 수정된 파일(BluetoothManager, AiViewModel 등)의 상태를 확인합니다.
* **`git add .`**: 변경된 모든 사항을 스테이징 영역에 추가합니다.
* **`git commit -m "branch_name"`**: 작업 내역을 로컬 저장소에 기록합니다.
* **`git push origin "branch_name"`**: 원격 브랜치로 코드를 업로드하여 집에서도 작업할 수 있도록 준비합니다.

---

## 2. 브랜치 전략 및 테스트 환경 구축 (Tracking & Python)
안드로이드의 로직을 파이썬 웹캠 환경에서 "재미삼아" 테스트하기 위해 브랜치를 넘나들며 사용한 흐름입니다.

* **`git switch [브랜치_이름]`**: 기존의 `checkout`을 대체하여 작업 브랜치로 안전하게 이동합니다.
* **`git switch -c [새_브랜치_이름]`**: 새로운 트래킹 전용 브랜치를 생성함과 동시에 해당 브랜치로 전환합니다.
* **`git merge feature/AI-16-android-AI`**: 안드로이드 브랜치에서 완성한 AI 로직(GestureClassifier 등)을 현재의 트래킹 브랜치로 병합합니다.

---

## 3. 환경 전이 및 협업 (Remote → Home)
퇴근 후 집 PC에서 사무실의 작업 내용을 그대로 이어받을 때 사용합니다.

* **`git pull origin "branch_name"`**: 원격 저장소의 최신 커밋 내역을 로컬로 가져와 동기화합니다.
* **`git log --oneline --graph --all`**: 현재 브랜치들의 병합 상태와 커밋 히스토리를 시각적으로 확인합니다.

---

## 4. 기타 유용한 명령어
* **`git diff`**: 코드 수정 사항을 커밋 전에 라인 단위로 상세히 비교합니다.
* **`git branch`**: 현재 로컬에 존재하는 브랜치 목록을 확인하고 내가 어디에 있는지 체크합니다.
